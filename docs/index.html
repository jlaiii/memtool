Features

Everything you need for memory analysis, scanning, and editing on Windows.

⚙

SIMD-Accelerated Scanner

NumPy-powered scans achieve 800+ MB/sec. 11 value types, 8 scan kinds, and full unaligned scanning across every byte offset.

★

Value Freezing at 250 Hz

Lock values in place with automatic VirtualProtectEx page unprotection. Write-back verification confirms every operation.

⇕

Pointer Chain Discovery

Find multi-level pointer chains rooted in static module addresses. Save chains to survive game restarts and share with others.

☉

Byte Signature Matching

IDA-style pattern scanning with wildcards. Create unique signatures around values that persist through game updates.

⌨

CLI for AI Integration

14 CLI commands with JSON output. Designed for Claude Code and other AI tools to perform autonomous memory analysis and editing.

☷

Smart Persistent Cache

Save address lists and pointer caches as JSON. Auto-rediscover values on re-attach using cached signatures and pointer chains.

Quick Start

Install, launch, attach, and scan in under a minute.

# Clone and install
git clone https://github.com/jlaiii/memtool.git
cd memtool
pip install -r requirements.txt

# Launch the GUI (triggers UAC for admin)
python run.py

# Or use the CLI for AI-assisted workflows
python -m memtool.cli processes --json
python -m memtool.cli attach --pid 1234
python -m memtool.cli scan --type int4 --value 100
python -m memtool.cli next --kind decreased
python -m memtool.cli write --address 0x7FF12345 --type int4 --value 999
python -m memtool.cli freeze --address 0x7FF12345 --type int4 --value 999

About

Project details, roadmap, and how to contribute.

Project

MemTool is an open-source Windows memory analysis tool built with Python and PyQt6. It is inspired by Cheat Engine and designed for single-player game modding, reverse engineering education, and memory forensics research.

Website: https://jlaiii.github.io/memtool

Repository: https://github.com/jlaiii/memtool

License & Restrictions

  • MIT License -- free to use, modify, and distribute
  • Windows only (uses Win32 API via ctypes)
  • Requires Administrator privileges for memory access
  • Intended for single-player games and educational use
  • Not for use with multiplayer games or software that prohibits memory modification

Technology Stack

  • GUI: PyQt6 with Catppuccin Mocha dark theme
  • Scanning: NumPy with SIMD vectorization
  • Memory I/O: Windows API via ctypes (ReadProcessMemory, WriteProcessMemory, VirtualProtectEx)
  • Module Enumeration: Toolhelp32Snapshot API
  • Persistence: JSON in ~/MemTool/

Roadmap

  • Debug register breakpoints (find what writes/accesses this address)
  • Dissect data structures / struct builder
  • Code injection and DLL injection
  • Kernel driver for hardware breakpoints and protected process access
  • Auto-assembler for scripting and code caves
  • Lua scripting engine
